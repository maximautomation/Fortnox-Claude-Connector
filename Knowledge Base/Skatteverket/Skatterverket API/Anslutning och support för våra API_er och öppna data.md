## Vanliga frågor och svar om API:er

### Vilken teknisk lösning använder Skatteverket?

Skatteverket använder idag olika tekniska plattformar för API:erna. Våra API:er bygger i första hand på REST över http och json med OAuth2 som teknik för auktorisation.

I Utvecklarportalen kan du läsa om vad som gäller för varje enskilt API. Där finns all dokumentation om API:et; allmän information, säkerhetsinformation, teknisk dokumentation och exempelkod som visar hur anrop och svar ska se ut. I Utvecklarportalen finns också information om de villkor och avtal som gäller för att få tillgång till API:et.

- [Utvecklarportalen Länk till annan webbplats.](https://www7.skatteverket.se/portal/apier-och-oppna-data/utvecklarportalen/)

### Vilka miljöer finns?

Varje API lanseras som komplett testtjänst (sandbox) och driftsatt tjänst.

### Komplett testtjänst (sandbox)

Komplett testtjänst är en testmiljö med dynamiska data. Du behöver ansöka om att få tillgång till dem genom att fylla i formulär på Skatteverkets webbplats. Du behöver även godkänna Skatteverkets användarvillkor. När vi har godkänt din förfrågan får du API-nycklar, client\_id och client\_secret. Du behöver dem för att anropa API:et. Nycklarna skickas via mejl. Testdata och URI till den kompletta testtjänsten finns för respektive API i Utvecklarportalen.

### Driftsatt tjänst

Driftsatt tjänst är API:et i produktion. För att få tillgång ska ditt företag uppfylla Skatteverkets krav för API:et och du ska skriva under ett avtal. Du kan ta del av avtal och villkor för respektive API i Utvecklarportalen. Fyll sedan i formuläret ”Begär tillgång till driftsatt tjänst”. När vi har godkänt din förfrågan och avtalen är signerade får du API-nycklar client\_id och client\_secret. De behöver du för att anropa API:et. Nycklarna skickas via mejl och sms. URI till den driftsatta tjänsten finns för respektive API i Utvecklarportalen.

- [Utvecklarportalen Länk till annan webbplats.](https://www7.skatteverket.se/portal/apier-och-oppna-data/utvecklarportalen/)

## Hur får jag använda API:erna?

Vi använder oss av en anslutningsprocess, avtal och tekniska säkerhetslösningar för att säkerställa att personrelaterad och sekretessbelagd information skyddas. För varje API finns det specifika krav som du behöver uppfylla för att få tillgång till API:et. De krav som Skatteverket ställer finns i Utvecklarportalen under respektive API.

- [Utvecklarportalen Länk till annan webbplats.](https://www7.skatteverket.se/portal/apier-och-oppna-data/utvecklarportalen/)

### Hur uppdaterar eller avregistrerar jag tillgången till ett API?

Kontakta oss genom att fylla i ett formuläret "Uppdatera eller avsluta befintligt API". Kontakta oss om API:er om du behöver uppdatera din API-applikation med ny information, såsom nytt redirect\_URI eller ny kontaktperson. Du kan även avregistrera tillgången till API:et via formuläret.

Fyll i de sex sista siffrorna i API-nycklarna, alltså ditt client\_id och client\_secret i formuläret. På så vis identifierar du dig på ett säkert sätt.

När vi har tagit emot ditt ärende gör vi förändringen och meddelar dig via mejl.

- [Uppdatera anslutning till ett API](https://www.skatteverket.se/omoss/digitalasamarbeten/anslutningochsupportforvaraapierochoppnadata/uppdateraanslutningtillapi.4.7c708f0e16bed42cd055c37.html)
- [Avsluta anslutning till API](https://www.skatteverket.se/omoss/digitalasamarbeten/anslutningochsupportforvaraapierochoppnadata/avslutaanslutningtillapi.4.7c708f0e16bed42cd055c59.html)

### Vad är ett API?

API:er (Application Programming Interface) är en teknik som gör det möjligt för två program att utbyta data med varandra. De API:er som Skatteverket erbjuder gör det möjligt för andra aktörer, till exempel företag, att kommunicera med våra system och på så vis använda Skatteverkets data i sina egna tjänster. Lär dig mer om API:er här:

- [Vad är ett API](https://www.skatteverket.se/omoss/digitalasamarbeten/omvaraapier/vadarettapi.4.96cca41179bad4b1aaa4b8.html)

### Vad är en redirect-URI?

En redirect-URI är en adress hos er som används för att ange vart anrop ska dirigeras tillbaka efter en lyckad autentisering. Du skickar med redirect-URI:n i anropet och den jämförs med en av de redirect-URI:er som du anmälde till oss i samband med registrering. På så vis kan vi skydda trafiken från att omdirigeras till andra främmande adresser efter autentiseringen.

Redirect-URI används i OAuth2.0-flödet Authorization Code Grant (ACG) som Skatteverket nyttjar för autentisering.

Formatet för en redirect-URI:  
https://domännamn\[:port\]/\[resurs\]

Läs mer om ACG:

- [Authorization Code Grant (ACG)](https://www.skatteverket.se/omoss/digitalasamarbeten/borjaanvandaapier/authorizationcodegrantacg.4.96cca41179bad4b1aa8f05.html)

## Vanliga frågor och svar om öppna data

### Hur får jag använda informationen?

Skatteverkets data består idag enbart av data där det inte går att urskilja enskilda personer. Datan är fri att använda utan några avtal eller avgifter.

### Lagar om vidareutnyttjande av information

Vidareutnyttjande av myndighetsinformation regleras i PSI-lagen. PSI står för Public Sector Information, alltså offentlig information.

- [Lag (2022:818) om den offentliga sektorns tillgängliggörande av data Länk till annan webbplats.](https://www.riksdagen.se/sv/dokument-och-lagar/dokument/svensk-forfattningssamling/lag-2022818-om-den-offentliga-sektorns_sfs-2022-818/)

Information för vidareutnyttjande omfattar aldrig information där privatpersoner eller företag kan identifieras direkt eller indirekt. Det regleras i de här lagarna:

- [EU:s dataskyddsförordning Länk till annan webbplats.](https://eur-lex.europa.eu/legal-content/SV/TXT/?qid=1525690211521&uri=CELEX:32016R0679)
- [Lag (2001:181) om behandling av uppgifter i Skatteverkets beskattningsverksamhet Länk till annan webbplats.](http://www.riksdagen.se/sv/dokument-lagar/dokument/svensk-forfattningssamling/lag-2001181-om-behandling-av-uppgifter-i_sfs-2001-181)

### I vilket format finns Skatteverkets öppna data?

Informationen lämnas i ett format som följer en öppen standard, främst csv (comma-separated values) och ett tillhörande API (application programming interface). För informationsmängder där andra format efterfrågats försöker vi tillhandahålla även dessa. Det kan vara till exempel excel eller txt.

Vi publicerar informationen i så maskinläsbar form som möjligt. På sikt vill vi tillgängliggöra informationen som ett API eller som länkad data.

### Vad är öppna data?

Öppna data är information som finns tillgänglig för vem som helst att använda, återanvända och dela med sig av, så att andra kan utveckla den och skapa nytta för fler. Skatteverkets öppna data är alltid på aggregerad nivå, alltså en sammanställning av många olika data. Det går inte att få ut data på individnivå via våra öppna data.
