Möt Susanne Bern, produktägare för systemlösningen Kontek Lön, och läs om hur Konteks API-idé blev verklighet samt varför hon anser att fler borde våga samarbeta med Skatteverket.

## Specialister på lön sedan 1934

Kontek är ett specialistbolag inom lön och har funnits med länge, ända sedan 1934. Susanne har jobbat där i 17 år och är idag produktägare för det webbaserade lönesystemet Kontek Lön. Systemet lämpar sig väl för API:er, de använder därför Skatteverkets API Fråga om skatteavdrag. Det var dock avsaknaden av ett API som blev startskottet för Konteks samarbete med Skatteverket.

## Nybilspriser – från manuellt lagrade filer till API på ett år

Vi backar några år. Kontek Lön innehöll en funktion som hjälpte arbetsgivare att automatiskt generera värde för bilförmån. Det här gjordes genom att hämta nybilspriser från Skatteverkets föreskrifter via de filer som kontinuerligt publicerades. Susanne behövde bevaka när de nya filerna kom för att sedan lägga in dem i Kontek Lön. Först därefter kunde Kontek leverera en ny programversion för att de nya bilarna skulle vara tillgängliga för användaren.

När Susanne såg att Skatteverket presenterade nybilspriser för bilar per tillverkningsår bland Skatteverkets öppna data, beslutade hon sig för att kontakta Skatteverket. Det hon efterfrågade fanns ju i princip där men problemet var att hon behövde kunna hämta uppgifter om bilar för flera tillverkningsår, inte bara det senaste.

### Vinsten med API:et

Susanne såg förbättringspotential och hörde därför av sig till Skatteverket med förslaget att man kanske kunde bygga ett API som samlar flera år. Responsen var positiv. Skatteverket hade förståelse för det här behovet.

– Nästan exakt ett år senare fanns API:et på plats, som komplement till de öppna data som redan fanns. Nu innehåller det här API:et nybilspriser för bilar med tillverkningsår 2013-2020. Vi byggde en API-integration mot Skatteverket som hämtar uppgifter om bilarna.

Vinsten blev tydlig direkt.

– Nu slipper vi bevaka Skatteverkets föreskrifter om nybilspriser vilket sparar oss tid, risken för manuella fel minskar avsevärt och processen blir även snabbare för våra kunder. Så fort Skatteverket har uppdaterat API:et med nya bilar blir de tillgängliga för Kontekts användare.

## Våga fråga och ge förslag på utveckling

Susanne vill uppmuntra andra programvaruleverantörer att faktiskt våga kontakta Skatteverket.

– Mitt bästa tips är att våga prova att samarbeta. Våga kom med förslag, det kommer falla i god jord. Ni kommer dessutom få en annan bild av Skatteverket. Det är inte en stor gigantisk myndighet att jobba med, det finns människor där bakom som är alerta och vill hjälpa till.

Susanne säger att hon är ödmjuk och tacksam för att hennes förslag blev verklighet men förvånad är hon egentligen inte.

– Om man läser på skatteverket.se så ser man ju tydligt att ni vill samarbeta. Det finns flera kontaktvägar in och allt ni pratar om handlar om att vi tillsammans kan skapa nytta för användarna. Jag tycker det är kanon.

## Integrera mera – men med sunt förnuft

Integrationer är väldigt hett på marknaden just nu, i alla fall om man lyssnar på vad våra kunder frågar om, säger Susanne. Och självklart, integrationer av olika slag skapar ofta stor kundnytta men Susanne påpekar att det ändå är en risk att öppna upp sina system mot andras.

– Man ska inte bygga integrationer bara för att bygga dem. De måste göra solklar nytta. För hur det än är så är det en sårbarhet att bygga in massa system i varandra.

Hon menar att det viktigaste är att välja med omsorg, inte slitas med i nyhetens behag.

– Om ett API är helt nytt följer ibland även en del barnsjukdomar. Det behöver man kanske ta höjd för innan man sätter igång. När jag känner att samarbetspartners teknik är stabil och att jag kan förlita mig på att resultatet som genereras är korrekt, först då börjar vi bygga.

## ”Vi ska vara idésprutor”

Skatteverket kan inte heller alltid veta vad som borde satsas på inom det här området, säger Susanne. Det är därför samarbetet är så viktigt.

– Jag är ingen utvecklare. Jag är en kravställare och produktägare. Mitt jobb är att tänka fritt kring vilka lösningar som skulle gynna både oss och våra kunder mest, sedan finns det utvecklare på Skatteverket som kan hjälpa mig avgöra vad som är praktiskt genomförbart.

Vi ska vara idésprutor så att ni får input på vad ni bör satsa på, fortsätter hon.

På frågan om hon skulle rekommendera andra att börja samarbeta med Skatteverket kommer svaret blixtsnabbt.

– Absolut! Jag är väldigt nöjd med hela processen.
