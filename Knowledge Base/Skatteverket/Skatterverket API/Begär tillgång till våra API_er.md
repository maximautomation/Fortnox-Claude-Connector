Hur du använder våra API:er kan skilja sig åt beroende på vilket API det gäller och för vilken typ av miljö. Du behöver också ansöka om tillgång via våra formulär. Ibland måste du även skaffa någon form av elektronisk legitimation och implementera en säkerhetslösning.

Riktade API:er samt API för Faderskapsutredning ansöker du om via särskilda formulär. Du hittar länkar till samtliga formulär och mer information på denna sida.

Du hittar alla Skatteverkets API:er på Utvecklarportalen.

- [Utvecklarportalen Länk till annan webbplats.](https://www7.skatteverket.se/portal/apier-och-oppna-data/utvecklarportalen?dataresurs=partner-api&dataresurs=riktat-api)

## Partner-API:er – kräver avtal för driftsatta tjänster

Våra partner-API:er riktar sig till myndigheter, organisationer och programvaruföretag. Du måste skriva under ett avtal för att få använda driftsatta tjänster. För kompletta testtjänster krävs inget avtal, men du måste fortfarande fylla i ett ansökningsformulär.

För många av våra API:er finns också speciella villkor och krav. Du hittar mer information under respektive API:s sida på Utvecklarportalen.

## Innan du begär tillgång till våra partner-API:er

Vilka uppgifter vi behöver från dig beror på vilket API du vill få tillgång till, och vilken säkerhetslösning som du kommer att använda.

Vi kan bland annat behöva uppgifter om:

- Personnummer eller organisationsnummer
- Namn och e-postadress till kontaktperson.
- Namn och e-postadress till den person som ska få API-nycklarna
- Information om firmatecknare eller den som signerar avtalet hos er
- Organisationslegitimation
- Redirect-URI.

Använd gärna e-postadresser som inte är kopplade till en specifik person, utan till en arbetsgrupp eller liknande.

Om du vill återanvända befintliga API-nycklar fyller du i applikations-id för applikationen som API-nycklarna varit kopplade till sen tidigare.

## Riktade API:er – kräver lagstöd och avtal

Vissa API:er har en begränsad målgrupp och ett annat anslutningsförfarande. Dessa kallar vi för ’riktade API:er’. För att få använda dem behöver du göra en överenskommelse med Skatteverket. När vi har tagit emot din begäran om anslutning till ett API så undersöker vi om alla förutsättningar finns på plats för att godkänna din ansökan.

Vi rekommenderar att du läser igenom våra sidor om avtalsprocessen innan du söker. Tänk på att ansöka i god tid! Det kan även vara bra att läsa informationen på sidorna "Om våra API:er" och "Börja använda API:er".

## Innan du begär tillgång till våra riktade API:er

För att få tillgång till våra riktade API:er behöver du fylla i och skicka in ett särskilt formulär. Därefter ser processen ut ungefär så här:

När Skatteverket har tagit emot din begäran om anslutning till ett API så undersöker Skatteverket om alla förutsättningar finns på plats för att godkänna din ansökan. I samband med detta kan Skatteverket behöva komma i kontakt med dig. Det är därför viktigt att du fyller i korrekta kontaktuppgifter i formuläret.

Om din begäran om anslutning godkänns behöver Skatteverket upprätta en överenskommelse med din organisation som bland annat reglerar kommunikationssätt, vad som gäller vid ändringar av API:et, hur ansvarsfördelningen ser ut och hur länge överenskommelsen är giltig.

När en överenskommelse har upprättats så tilldelas din organisation API-nycklar i form av “client\_id” och “client\_secret”. API-nycklar är unika för er programvara och bör hanteras på ett säkert sätt så att inte obehöriga får åtkomst till dem.  
För ökad säkerhet skickas API-nycklarna uppdelade:

- en nyckel skickas via e-post
- en nyckel skickas via sms.
